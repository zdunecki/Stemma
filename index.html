Semma

The dynamic checkbox tree

build on object data

    var data = [
        {child:'o1',text:'Option 1'},
        {mineparent:'o1',child:'o1-o1',text:'Option 1.1'},
        {mineparent:'o1',child:'o1-o2',text:'Option 1.2'},
        {mineparent:'o1-o2',child:'o1-o2-o1',text:'Option 1.2.1'},
        {mineparent:'o1-o2',child:'o1-o2-o2',text:'Option 1.2.2'},
        {child:'o2',text:'Option 2'},
        {mineparent:'o2',child:'o2-o1',text:'Option 2.1'},
        {mineparent:'o2',child:'o2-o2',text:'Option 2.2'},
        {mineparent:'o2-o2',child:'o2-o2-o1',text:'Option 2.2.1'},
        {mineparent:'o2-o2',child:'o2-o2-o2',text:'Option 2.2.2'}
      ];

build on DOM

<div id="mount">
    <input type="checkbox" data-child="o1"><label for="mount-o1">Option 1</label> <br>
        <input type="checkbox" data-mine-parent="o1" data-child="o1-o1"><label for="mount-o1-o1">Option 1.1</label> <br>
        <input type="checkbox" data-mine-parent="o1" data-child="o1-o2"><label for="mount-o1-o2">Option 1.2</label> <br>
            <input type="checkbox" data-mine-parent="o1-o2" data-child="o1-o2-o1"><label for="mount-o1-o2-o1">Option 1.2.1</label> <br>
            <input type="checkbox" data-mine-parent="o1-o2" data-child="o1-o2-o2"><label for="mount-o1-o2-o2">Option 1.2.2</label> <br>
    <input type="checkbox" data-child="o2"><label for="mount-o2">Option2</label> <br>
        <input type="checkbox" data-mine-parent="o2" data-child="o2-o1"><label for="mount-o2-o1">Option 2.1</label> <br>
        <input type="checkbox" data-mine-parent="o2" data-child="o2-o2"><label for="mount-o2-o2">Option 2.2</label> <br>
            <input type="checkbox" data-mine-parent="o2-o2" data-child="o2-o2-o1"><label for="mount-o2-o2-o1">Option 2.2.1</label> <br>
            <input type="checkbox" data-mine-parent="o2-o2" data-child="o2-o2-o2"><label for="mount-o2-o2-o2">Option 2.2.2</label> <br>
</div>

Usage

var Options = new Stemma(mountElement,{options});

Parameters

  • mountElement(DOM element) - required parameter - to bind/find checkboxes
  • options(Object) - optional parameter
    • data - bind object data to mountElement and convert as DOM elements. Data properties:
      • child - any string
      • mineparent - look for each child property to indentify relation child-parent. If mineparent equal any child property of object means that mineparent checkbox is a children of child checkbox. Remember, If you set autorelation as true to keep relation child-parent then properties merges of mineparent and child have to equal mineparent property
