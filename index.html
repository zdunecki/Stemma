Example

Generation

New generation syntax

To build new generation we need dash syntax. The dash build new generation beetwen mineparent and child property or data-child and data-mine-parent attributes if we build DOM manually,

First generation

First generation start with child property and dont need mineparent becouse it is independent of all posterity.

Build new generation

Use new generation syntax to build tree relationship. Example:

{child:father,text:'I am father'}, {mineparent:'father',child:'father-children1',text:'I am children'}, {mineparent:'father-children1',child:'father-children1-children1',text:'I am children of father children'}, {mineparent:'father',child:'father-children2',text:'I am children 2'},

Father contains all objects becouse each mineparent of objects have father generation, but first father's children contain only one object becouse only one object have father-children1 generation.

Usage

<script type="text/javascript"  src="build/stemma.js"></script>

var Options = new Stemma(mountElement,{options});

Parameters

  • mountElement(DOM element) - required parameter - to bind/find checkboxes
  • options(Object) - optional parameter
    • data - bind object data to mountElement and convert as DOM elements. Data properties:
      • child - It has a unique name for each object. If child is a new generation then child name is father child property and any name maintaining new generation syntax.
      • mineparent - look for each child property to indentify relation child-parent. If mineparent equal any child property of object means that mineparent checkbox is a children of this child checkbox. Remember to use new generation syntax to create tree relationship.
      • text - text for label element
